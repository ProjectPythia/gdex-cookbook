Introduction to the GDEX API

Overview¶

The GDEX REST API provides programmatic access to dataset discovery, metadata, subset requests, and file downloads. This notebook introduces the API conventions, authentication, and the standard response envelope.

  1. Understand the base URL and endpoint structure

  2. Learn the standard JSON response envelope

  3. Authenticate with a token or Basic auth

  4. Make your first API call

API Conventions¶

All endpoints live under https://gdex.ucar.edu/api/. Dataset IDs use the format d###### (six digits). Every successful response follows this envelope:

{
  "status": "ok",
  "http_response": 200,
  "error_messages": [],
  "contact": "datahelp@ucar.edu",
  "data": { ... }
}

Authentication¶

Many read endpoints are public. Subset/download endpoints require authentication via:

  • Token: ?token=<your_token> — get yours at /accounts/profile/

  • Basic auth: requests.get(url, auth=("username", "password"))

Summary¶

The GDEX API follows a consistent REST pattern: GET https://gdex.ucar.edu/api/<endpoint>/. Public endpoints need no auth; subset and download endpoints require a token from your GDEX profile.

Transferring GDEX Data with Globus

Authors
Affiliations
NSF NCAR

NSF NCAR

Overview¶

Globus enables fast, reliable transfers of large datasets from GDEX to your HPC system (or to a personal computer using Globus Connect Personal).

With a completed GDEX subset request, you can start a transfer directly through the GDEX API. See the GDEX API Globus documentation for endpoint details.

In this notebook, you will:

  1. Understand what a Globus endpoint is

  2. Prepare a Globus account and destination endpoint

  3. Initiate a transfer using the Globus web UI

  4. Monitor transfer status in the Globus web UI

Prerequisites¶

Globus accountNecessaryAnd a Globus endpoint at your destination

Globus End Point¶

A Globus endpoint is a storage location that Globus can read from or write to during a transfer.

For this workflow:

  • Source endpoint: the GDEX-managed endpoint that contains the GDEX datasets.

  • Destination endpoint: your endpoint (for example, an HPC system, campus storage, or Globus Connect Personal on your computer).

Each endpoint has a unique identifier called an Endpoint UUID.

Tip

  • Verify your destination endpoint is visible in app.globus.org.

  • Make sure the endpoint is active/online and you have write permission to the target path.

  • Keep the UUID exact (including dashes), since it is required in the transfer request.

Globus Account and Endpoint¶

You can create a Globus account using one of several identity providers, including:

  • ORCID iD

  • Google

  • GitHub

  • Institutional login (if supported by your organization)

After signing in, you can add your personal computer as a destination endpoint by installing Globus Connect Personal.

  • Installation guide: https://docs.globus.org/globus-connect-personal/install/

  • Download page: https://www.globus.org/globus-connect-personal

Once Globus Connect Personal is installed and running, your personal endpoint will appear in Globus and can be used as the transfer destination.

Initiate Transfer¶

Once the source (NSF NCAR GDEX Dataset Archive) and destination endpoints are configured in the Globus File Manager, you can proceed with the transfer.

Navigate to the desired folder on the GDEX endpoint and select the files or directories you wish to transfer.

Click the Start button to initiate the transfer. The transfer will run in the background — you do not need to keep your browser open. You can monitor progress in the Activity panel on the left, and Globus will send you an email notification when the transfer is complete.

GDEX Globus Service¶

You can access the NSF NCAR GDEX Dataset Archive endpoint in two ways:

  1. Search directly in the Globus web UI

  2. Use dataset-specific links from the GDEX web portal’s data access tab

Summary¶

Globus transfers are the recommended method for moving large GDEX datasets to HPC systems. The GDEX API can initiate the transfer directly from a completed request index.

Overview¶

The GDEX THREDDS Data Server (TDS) provides remote access to datasets via OPeNDAP, NCSS, WMS, and other standard protocols — without downloading files. This notebook introduces TDS concepts and shows how to navigate the catalog.

  1. Understand what TDS provides

  2. Browse the GDEX TDS catalog

  3. Identify available data services for a dataset

Accessing TDS via the Web Browser¶

The GDEX TDS catalog can be explored interactively through any web browser — no Python or programming required. This section walks through the key steps.

Step 1: Open the GDEX TDS Catalog¶

Navigate to the GDEX TDS catalog root in your browser:

https://tds.gdex.ucar.edu/thredds/catalog/catalog.html

You will see a list of top-level dataset collections. Each entry corresponds to a GDEX dataset identified by its dataset ID (e.g., d010049).

Step 2: Browse to a Dataset¶

Click a dataset ID folder (e.g., d010049) to enter its sub-catalog. Sub-catalogs are organized by year or other time partitions. Click through until you reach a directory containing individual files.

Step 3: View Available Services for a File¶

Click on an individual file name to open its dataset page. The page lists all access services (OPeNDAP, NCSS, WMS, HTTP download, etc.) with direct links — you can copy any URL for use in Python or paste it into a browser to download.

Accessing TDS via Siphon — Programmatic Approach¶

Siphon

Siphon is a Python client for THREDDS and other remote data services. It can:

  • browse TDS catalogs and sub-catalogs

  • inspect available datasets and access URLs

  • query metadata and service endpoints

  • work with remote data through OPeNDAP, NCSS, and related services

  • integrate with xarray for analysis of remote NetCDF data

Use it when you want to discover and access datasets without downloading full files first.

Services available for this dataset¶

The dataset mercatorglorys12v1_gl12_mean_19930101_R19930106.nc exposes multiple access protocols through the GDEX THREDDS Data Server.

  • OpenDAP — Remote data access and subsetting over HTTP; commonly used with tools like xarray for analysis without downloading full files.

  • NetcdfSubset — NetCDF Subset Service (NCSS) for requesting specific variables, time ranges, vertical levels, and geographic regions.

  • CdmRemote — Binary remote access protocol for CDM-compatible clients; useful for efficient programmatic data retrieval.

  • CdmrFeature — Feature-oriented CDM remote access for structured scientific data types (for example, grids and point features).

  • JupyterNotebook — Launch/open notebook-based workflows connected to the dataset in an interactive environment.

  • WCS — Web Coverage Service for retrieving geospatial raster/coverage data with spatial and dimensional subsetting.

  • WMS — Web Map Service for requesting rendered map images of dataset layers for visualization.

  • HTTPServer — Direct file download access over HTTP.

  • ISO — ISO-standard metadata representation for cataloging and discovery.

  • NCML — NetCDF Markup Language view describing dataset structure and metadata in XML form.

  • UDDC — Unified Data Discovery Convention metadata endpoint for interoperable dataset discovery.

These services support different workflows, from quick map visualization (WMS) to analysis-ready subset extraction (NetcdfSubset/OpenDAP) and metadata discovery (ISO/NCML/UDDC).
